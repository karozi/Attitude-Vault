## Project-Specific Rules

### When Adding Content
- **New prompt** → Determine category → `prompts/[category]/[prompt-name].md`
- **New guide** → `guides/[guide-name].md`
- **New automation** → `automations/[automation-name]/`
- **Always commit and push** after adding content

### File Naming
- All files: kebab-case (e.g., `system-prompts-best-practices.md`)
- Folders: kebab-case (matching screenshot structure)

### Content Standards
- Prompts: Include usage examples and expected output
- Guides: Practical, actionable content
- Automations: Include README with setup instructions

## Relationship to Creator-Lab
Attitude-Vault is complementary to creator-lab:
- **Creator-Lab**: Full agents, skills, workflows (executable)
- **Attitude-Vault**: Prompts, guides, tools (instructional)
